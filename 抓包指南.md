# 🔍 寿司郎排队 API 抓包指南 (Windows)

## 总览

寿司郎的排队系统没有公开 API, 我们需要从微信小程序中抓取。有三种方案:

| 方案 | 难度 | 需安装的软件 |
|------|------|------------|
| **A: PC微信 + 系统代理** | ⭐⭐ | Python + mitmproxy |
| **B: PC微信 + Proxifier** | ⭐⭐⭐ | Python + mitmproxy + Proxifier |
| **C: 安卓模拟器** | ⭐⭐⭐⭐ | 模拟器 + mitmproxy + Frida |
| **D: 反编译小程序包** | ⭐⭐⭐⭐⭐ | 反编译工具链 |

---

## 方案A: PC 微信 + 系统代理 (推荐先试这个)

### 步骤 1: 安装 mitmproxy

打开终端 (PowerShell 或 CMD):

```bash
pip install mitmproxy
```

### 步骤 2: 启动抓包

在项目目录下:

```bash
cd D:\Software\SOS\sushiro-monitor
mitmweb -s capture_proxy.py
```

这会:
- 启动代理服务器在 **127.0.0.1:8080**
- 打开一个 Web 界面在 http://localhost:8081 (可在浏览器里查看抓到的请求)
- 自动将寿司郎相关的 API 保存到 `captured_apis/` 目录

### 步骤 3: 安装 mitmproxy 证书

1. 浏览器打开 **http://mitm.it**
2. 下载 Windows 证书
3. 双击安装 → **受信任的根证书颁发机构**

### 步骤 4: 设置系统代理

**Windows 10/11:**

```
设置 → 网络和 Internet → 代理 → 手动设置代理
  使用代理服务器: 开
  地址: 127.0.0.1
  端口: 8080
  保存
```

> ⚠️ 抓包完成后记得关掉系统代理, 否则正常上网会受影响!

### 步骤 5: 打开寿司郎小程序

1. 打开 PC 微信
2. 搜索 "寿司郎" → 进入官方小程序
3. 进入排队页面, 查看排队状态
4. 看终端输出, 找 API 请求

### 步骤 6: 查看结果

打开 `captured_apis/` 目录, 按时间排序, 找到排队的 API 请求。

JSON 文件里包含完整的请求 URL、请求头、请求体和响应体。

## 方案B: PC微信 + Proxifier (如果方案A不行)

微信小程序的网络请求有时不走系统代理, 需要用 Proxifier 强制代理。

### 额外准备

- 下载 Proxifier: https://www.proxifier.com/ (有31天试用)
- 或使用免费的 **Clash Verge** 开启 TUN 模式

### Proxifier 设置

```
1. Profile → Proxy Servers → Add:
   Address: 127.0.0.1
   Port: 8080
   Protocol: HTTPS

2. Profile → Proxification Rules → Add:
   Name: WeChat Capture
   Applications: WeChatApp.exe; WeChatAppEx.exe; WechatBrowser.exe
   Action: Proxy HTTPS 127.0.0.1

3. 确保规则在列表顶部 (优先级最高)
```

### 其他步骤同方案A

## 方案C: 安卓模拟器 (前两种不行的备选)

如果 PC 微信抓不到, 可以用安卓模拟器:

1. 安装 **雷电模拟器** 或 **MuMu模拟器**
2. 模拟器内安装微信 + 寿司郎小程序
3. 模拟器设置 WiFi 代理 → 指向你电脑的 IP + 8080端口
4. 安装 mitmproxy 证书到模拟器的系统证书区 (需要 root 或使用 Magisk + TrustMeAlready)

---

## 方案D: 反编译 (终极手段)

如果代理抓包全部失败, 可以从微信小程序源文件中直接读取 API 信息:

1. 在 PC 微信打开一次寿司郎小程序 (确保加载过)
2. 进入目录: `C:\Users\你的用户名\Documents\WeChat Files\Applet\`
3. 找到寿司郎小程序的 AppID 对应的文件夹
4. 用 `UnpackMiniApp.exe` 解密 `.wxapkg` 文件
5. 用 `unveilr` 反编译为源码
6. 在源码中搜索: `request`, `api`, `queue`, `waiting` 等关键词

## ❓ 常见问题

### Q: 抓到一堆请求, 哪个是排队 API?

**特征:**
- URL 中包含 `queue` / `waiting` / `number` / `line` / `排队` 等关键词
- 返回 JSON 格式
- 响应中包含数字 (叫号号码)
- 请求可能带有 `Authorization` 头

**快速验证方法:**
复制请求 URL 和 token, 在浏览器里用 fetch 或 curl 试一下, 看返回是不是排队数据。

### Q: 没看到寿司郎相关请求?

- 确认在小程序里真正进入了排队页面 (不只是首页)
- 确认代理设置正确 (在浏览器里打开 http://mitm.it 验证)
- 尝试刷新小程序页面
- 如果完全没抓到任何请求, 说明代理没生效, 换方案B

### Q: Token 多久过期?

一般几小时到一天。如果仪表盘突然报错 401/403, 需要重新抓包获取新 token, 更新 config.json。

### Q: 抓到的是加密数据?

部分小程序会加密 API 通信。如果响应体是乱码:
1. 检查是否是 gzip/br 压缩 (mitmproxy 一般会自动解压)
2. 如果确实是加密的, 需要从反编译的源码中找到解密逻辑
3. 这种情况比较少见, 寿司郎大概率不加密
